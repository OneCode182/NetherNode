# Decision Log Protocol

Log durable decisions in:

- `memory/decisions.md` for architecture/product choices.
- `memory/patterns.md` for reusable implementation patterns.
- `memory/mistakes.md` for avoidable pitfalls.
- `memory/module-status.md` for subsystem status.
- `tasks/active/*.task.md` for current execution detail.

Keep entries short, dated, and evidence-backed.
